LEP-001. Способы исполнения кода.

Введение

В процессе разработки рассамтриваются несколько вариантов исполнение кода, написанном на Lumen. Для этого требуется определить сферы языка и вместе с тем целевую платформу.

Цели языка и целевая платформа.

Предполагается использование Lumen как языка написания скриптов общего назначения для Windows и веб-страниц. Выбор Windows сделан в силу того, что стандартная реализация языка основана на платформе .NET Framework 4.0, однако нет серьёзных препятствий, чтобы не добавить поддержку MacOS и Linux, используя .NET Core в будущем.

Вопросы компиляции и интерпретации

Язык был сделан интерпретируемым так как на платформе .NET уже существует компилируемый язык функционального программирования - F#. В то же время, из интерпретируемых языков наибольшее распространение получили лишь IronPython и IronRuby, которые, однако, не являются функциональными.

Как интерпретировать на .NET

Было принято простое решение - строить дерево выражений и исполнять его напрямую, что позволяет исполнять код очень динамично. Конечно, скорость исполнения не велика и это может являться проблемой, которая в данный момент не решена, однако существует несколько возможных решений:

  • Создание более простого императивного языка, который будет интерпретироваться быстрее. Крайний случай - написание виртуальной машины (возможно использование проекта HIL)
  • Использование самой виртуальной машины .NET
  • Создание транспилятора в IronRuby или IronPython. В этом случае нам достаточно будет реализовать достаточно простой транспайлер в один из этих языков, чтобы получить их эффективность выполнения. Смотреть LEP-003. Проект Ferrum.
  • Создание транспилятора в JavaScript и его использование в связке с JavaScript.NET, Смотреть LEP-002. Проект Photon.
