Почему Klischee?

Простота

Klischee это язык, который осилит даже полный долбаёб!

Высокоуровневость

Для того, чтобы писать на Klischee не нужно знать тонкости устройства компьютера - он использует абстракции!

Мультипарадигменность

На Klsichee можно писать сразу в нескольких стилях: объектно-ориентированном, функциональном, процедурном...

Интеграция с .NET

Klischee можно легко использовать в своих .NET-приложениях. Он может интегрироваться с любым .NET языком: C#, F#, C++/CLI, PascalABC.NET

Современность

В Klischee реализованы самые современные средства программирования: лямбда-выражения, перегрузка операций, сборка мусора, интерфейсы, свойства, явная типизация и многое другое!

Кроссплатформенность

Klischee можно использовать на любой платформе с установленным .NET Framework 4 и выше

Динамика

Klischee является очень динамичным языком: он обладает динамической типизацией, создавать типы можно во время исполнения программы, можно расширить уже существующие типы и модули

Большая библиотека

Для Klischee имеется большое количество модулей, написанных на .NET языках, например, GDI - модуль для работы с графикой или Web - для работы с сетью.

GitHub FAQ

Всё имеет значение

Любая конструкция языка может вернуть какое-либо значение

a = 5                           #=> 5
b = if a < 8: 9 else: 1 #=> 9
c = if b < a: 10            #=> null
IO << { a + b }              #=> 14

Функции являются объектами первого порядка

В языке нет различий между функциями и данными - вы можете передавать функции в другие функции, сохранять функции в переменные и многое другое

fun f(x) x * x
fun app(function, argument) {
 return function(argument)
}
puts(app(f, 6)) #=> 36

Прототипное ООП

Klischee имеет как прототипное, так и основанное на классах ООП 

#Создаём новый расширяемый объект.
a = new { x = 5 }
# Создаём еще один объект и 
# наследуем его от объекта a
b = new { y = 8 } : a
puts(b.x) #=> 5
