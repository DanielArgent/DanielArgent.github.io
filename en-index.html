Why Klischee?

Easy

Klischee is a language which can overpower even a full dolbaeb!

High level

In order to write on Klischee not need to know the intricacies of the computer device - it uses abstraction!

Multiparadigmatic

On Klsichee you can write in several styles: object-oriented, functional, procedural...

Integrated with .NET

Klischee you can easily use in their .NET applications. It can integrate with any .NET language: C#, F#, C++/CLI PascalABC.NET

Modernity

Klischee implemented in the most modern means of programming: lambda expressions, operator overloading, garbage collection, interfaces, properties, explicit typing, and more!

Cross-platform

Klischee can be used on any platform with installed .NET Framework 4 and above

Dynamics

Klischee is a very dynamic language, it has dynamic typing, create types during program execution, you can extend existing types and modules

A large library

For Klischee has a large number of modules written in .NET languages, for example, GDI is a module for working with graphics or Web to work with the network.

All that matters

Any language construct can return any value

a = 5                           #=> 5
b = if a < 8: 9 else: 1 #=> 9
c = if b < a: 10            #=> null
IO << { a + b }              #=> 14

Functions are objects of the first order

In language there is no distinction between functions and data - you can pass functions to other functions, store functions in variables and much more

fun f(x) x * x
fun app(function, argument) {
 return function(argument)
}
puts(app(f, 6)) #=> 36

Прототипное ООП

Klischee имеет как прототипное, так и основанное на классах ООП 

#Создаём новый расширяемый объект.
a = new { x = 5 }
# Создаём еще один объект и 
# наследуем его от объекта a
b = new { y = 8 } : a
puts(b.x) #=> 5
